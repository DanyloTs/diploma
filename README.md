# Система моніторингу транспортного потоку на основі комп'ютерного зору

Дипломна робота присвячена порівняльному аналізу сучасних моделей детекції об'єктів для задачі моніторингу транспортного потоку. У проєкті реалізовано детекцію та відстеження транспортних засобів (автомобілі, автобуси, вантажівки) у відеопотоці, а також виявлення порушень смуг руху.

Порівнюються чотири моделі: YOLOv8n, YOLOv11n, RT-DETR-L та Faster R-CNN — за метриками mAP, точності, повноти, F1-score та швидкості обробки (FPS).

## Структура проєкту

```
diploma/
├── data/                        # Датасет та результати оцінювання
│   ├── coco/                    # COCO 2017 та CSV-результати моделей
│   └── video.mp4                # Відео з трафіком для демонстрації
├── notebooks/                   # Jupyter-ноутбуки для аналізу
│   ├── dataset_exploration.ipynb
│   ├── yolov8_evaluation.ipynb
│   ├── yolov11_evaluation.ipynb
│   ├── rtdetr_evaluation.ipynb
│   ├── faster_rcnn_evaluation.ipynb
│   ├── model_comparison.ipynb
│   └── analytics.ipynb
├── src/                         # Вихідний код додатку
│   ├── config.py                # Конфігурація (шляхи, розміри, лінії)
│   ├── detector.py              # Обгортка для детектора об'єктів
│   ├── tracker.py               # Відстеження та виявлення порушень
│   └── utils.py                 # Допоміжні функції
├── main.py                      # Головний застосунок у реальному часі
└── requirements.txt             # Залежності проєкту
```

## Вимоги

- Python 3.10 або новіший
- pip
- (Опційно) GPU з підтримкою CUDA для прискорення inference

## Встановлення

### 1. Клонування репозиторію

```bash
git clone <url-репозиторію>
cd diploma
```

### 2. Створення віртуального середовища

**macOS / Linux:**
```bash
python3 -m venv .venv
source .venv/bin/activate
```

**Windows:**
```bash
python -m venv .venv
.venv\Scripts\activate
```

### 3. Встановлення залежностей

```bash
pip install -r requirements.txt
```

> Встановлення може тривати кілька хвилин через розмір PyTorch та Ultralytics.

## Запуск

### Головний застосунок (відстеження в реальному часі)

Запускає обробку відео з детекцією транспортних засобів, відстеженням перетинань лінії та виявленням порушень смуг:

```bash
python main.py
```

Після запуску можна інтерактивно задавати лінії мишею прямо у вікні відео.

### Jupyter-ноутбуки

Запуск Jupyter для аналізу датасету, оцінювання моделей та порівняння результатів:

```bash
jupyter notebook
```

Відкрийте потрібний ноутбук з папки `notebooks/`:

| Ноутбук | Опис |
|---------|------|
| `dataset_exploration.ipynb` | Аналіз датасету COCO 2017 |
| `yolov8_evaluation.ipynb` | Оцінювання моделі YOLOv8n |
| `yolov11_evaluation.ipynb` | Оцінювання моделі YOLOv11n |
| `rtdetr_evaluation.ipynb` | Оцінювання моделі RT-DETR-L |
| `faster_rcnn_evaluation.ipynb` | Оцінювання моделі Faster R-CNN |
| `model_comparison.ipynb` | Порівняння всіх моделей |
| `analytics.ipynb` | Поглиблений аналіз метрик |

## Моделі

Моделі завантажуються автоматично при першому запуску ноутбука або застосунку. Файли `.pt` для YOLO-моделей зберігаються у кореневій директорії проєкту після першого завантаження.

| Модель | Розмір | FPS | mAP@0.5 |
|--------|--------|-----|---------|
| YOLOv8n | 6.2 MB | 39.9 | 0.476 |
| YOLOv11n | 5.4 MB | 40.7 | 0.500 |
| Faster R-CNN | 159.8 MB | 1.8 | 0.584 |
| RT-DETR-L | 63.4 MB | 2.6 | 0.686 |

## Датасет

Для оцінювання моделей використовується датасет **COCO 2017** (валідаційна вибірка). Датасет завантажується автоматично через бібліотеку `fiftyone` при першому запуску відповідного ноутбука.
